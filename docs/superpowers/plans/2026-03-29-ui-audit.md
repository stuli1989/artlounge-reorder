## Task 7: Recalculation Audit — Fix Missing Triggers

**Files:**
- Modify: `src/api/routes/skus.py` — add recalc on intent change
- Modify: `src/api/routes/suppliers.py` — add recalc on lead_time change

- [ ] **Step 1: Fix `update_reorder_intent` in skus.py**

The `update_reorder_intent` endpoint (line ~398) updates the intent but does NOT trigger recalculation. Intent affects status (must_stock forces minimum Reorder, do_not_reorder suppresses qty).

Add `BackgroundTasks` to the endpoint and trigger targeted recompute:

```python
from fastapi import BackgroundTasks
from engine.targeted_recompute import run_targeted_recompute

@router.patch("/skus/{stock_item_name}/reorder-intent")
def update_reorder_intent(
    stock_item_name: str,
    req: ReorderIntentUpdate,
    background_tasks: BackgroundTasks,
    user: dict = Depends(require_role("purchaser")),
):
    with get_db() as conn:
        with conn.cursor() as cur:
            cur.execute(...)
            row = cur.fetchone()
            if not row:
                raise HTTPException(404, "Stock item not found")
        conn.commit()

    # Trigger recompute for this SKU's brand
    def _recompute():
        with get_db() as conn:
            # Find the brand for this SKU
            with conn.cursor() as cur:
                cur.execute("SELECT category_name FROM stock_items WHERE name = %s", (stock_item_name,))
                cat_row = cur.fetchone()
            if cat_row:
                run_targeted_recompute(conn, [cat_row["category_name"]])
    background_tasks.add_task(_recompute)

    return dict(row)
```

Check if `run_targeted_recompute` is the right function by reading `src/engine/targeted_recompute.py` briefly. If it takes different args, adjust.

- [ ] **Step 2: Fix `update_supplier` in suppliers.py**

Currently only triggers recalc when `buffer_override` changes (line 110: `buffer_changed = "buffer_override" in sent_fields`). But changing `lead_time_default`, `lead_time_sea`, `lead_time_air`, or `typical_order_months` also affects reorder calculations.

Change the trigger condition to include lead time and coverage fields:

```python
recalc_fields = {"buffer_override", "lead_time_default", "lead_time_sea", "lead_time_air", "typical_order_months"}
needs_recalc = bool(recalc_fields & sent_fields.keys())
```

Then use `needs_recalc` instead of `buffer_changed`:
```python
if needs_recalc:
    background_tasks.add_task(_recalc_buffers)
```
